# 소프트웨어 요구사항 명세서 (SRS)
## TitleGen — 클로드 API 연동 제목 자동생성 서비스

---

## 1. 서론 (Introduction)

### 1.1 목적 (Purpose)
본 문서는 TitleGen 서비스 개발을 위한 요구사항을 정의한다.
콘텐츠 마케팅 초보자나 글쓰기 초보자가 짧은 시간 안에 좋은 제목을 만들 수 있도록, Claude API를 단계적으로 활용하는 웹 서비스의 설계·구현·테스트 과정에서 참조한다.

### 1.2 범위 (Scope)
- **사용자:** 콘텐츠 마케팅 초보자, 상품 제목을 빠르게 만들고 싶은 1인 판매자 (1인 사용, 관리자 없음)
- **기능:** 단계별 데이터 입력 → Claude API 3단계 호출 → 제목 생성 → 피드백 루프
- **목표:** 사용자가 3분 이내에 투자 대비 좋은 마케팅 제목을 완성

### 1.3 정의 및 약어
- **API:** Application Programming Interface
- **Claude API:** Anthropic사의 Claude 언어모델 API
- **시스템 프롬프트:** 사용자에게 노출되지 않고 자동으로 Claude에 전달되는 사전 정의 지시문
- **컨텍스트 누적:** 이전 Claude 응답을 다음 호출의 대화 이력에 포함하여 전달하는 방식
- **피드백 루프:** 사용자가 개선 사항을 입력하여 제목을 재생성하는 반복 과정

### 1.4 참조 문서
- PRD: `prd/run-001/prd.md`
- 네이버 데이터랩 쇼핑인사이트: https://datalab.naver.com/shoppingInsight/sCategory.naver

## 2. 전체 설명 (Overall Description)

### 2.1 제품 관점 (Product Perspective)
본 시스템은 외부 서버에 배포되며, 사용자는 웹 브라우저를 통해 접근한다.

```
[사용자 브라우저]
    ↓ 텍스트 입력 + 버튼 클릭
[프론트엔드 웹앱]
    ↓ HTTP POST
[백엔드 서버]  ← API 키 보관
    ↓ 시스템 프롬프트 + 사용자 입력 누적 결합
[Claude API (Anthropic)]
    ↓ 응답
[백엔드 서버]
    ↓
[프론트엔드 — 결과 텍스트박스에 표시]
```

### 2.2 제품 기능 요약 (Product Functions)

| Phase | 단계 | 내용 |
|-------|------|------|
| 1 | 데이터 수집 | 콘텐츠 텍스트, 상품명, 타겟 특성 입력 + 데이터랩 안내 |
| 2 | 타겟 분석 | Claude API #1 → 마케팅 타겟 보고서 생성 |
| 3 | 제목 생성 | Claude API #2 → 15자 미만 제목 후보 생성 |
| 4 | 제목 차별화 | Claude API #3 → 네이버 상위노출 대비 차별화 |
| 5 | 피드백 루프 | 사용자 피드백 반영 재생성 (최대 2회) + 완료 처리 |

### 2.3 사용자 특성 (User Characteristics)
- 1인 사용자, IT 지식 낮음, 직관적 UI 필요
- 관리자 기능 없음

### 2.4 제약 조건 (Constraints)
- 운영 환경: 외부 서버 배포 (인터넷 접속 가능), HTTPS 적용 필수
- Claude API 키: 서버 사이드 환경변수로 보관, 클라이언트 미노출
- 피드백 루프: 세션당 최대 2회
- 복사 버튼 없음 — 결과는 텍스트박스로만 표시

### 2.5 가정 및 의존성 (Assumptions and Dependencies)
- 네트워크 연결이 안정적일 것을 가정
- Claude API (claude-sonnet-4-6 또는 동급 최신 모델) 사용
- 백엔드: Python(Flask/FastAPI) 또는 Node.js 경량 서버
- 결과 히스토리는 세션 내에서만 유지, 영구 저장 없음

### 5.3 통신 인터페이스

## 6. 시스템 아키텍처 개요

```
[사용자] → [웹 브라우저] → [프론트엔드]
                               ↓ HTTP POST (입력 데이터)
                          [백엔드 서버]
                               ↓ API 호출 (시스템 프롬프트 + 누적 컨텍스트)
                          [Claude API]
                               ↓ 제목/보고서 응답
                          [백엔드 서버]
                               ↓
                          [프론트엔드 — 텍스트박스 결과 표시]
```

- Claude API 호출: 총 최대 5회/세션 (고정 3회 + 피드백 최대 2회)
- 대화 컨텍스트: 단일 스레드로 누적 전달

기술스택: steamlit, python, fastapi 사용
