# TitleGen — 클로드 API 연동 제목 자동생성 서비스

## 1. 제품 개요

### 목적
콘텐츠 마케팅 초보자나 글쓰기 초보자가 짧은 시간 안에 투자 대비 좋은 제목을 뽑아낼 수 있도록, Claude API를 3단계로 활용하는 제목 자동생성 웹 서비스를 제공한다.

### 핵심 가치
- **구조화된 흐름:** 데이터 수집 → 타겟 분석 → 제목 생성 → 차별화 → 피드백 루프의 단계적 프로세스
- **품질:** 상품명 우선 배치, 15자 미만, SNS 어조 등 사전 최적화된 프롬프트 자동 결합
- **시장 연결:** 네이버 데이터랩 쇼핑인사이트로 실제 타겟 특성 확인 유도

### 대상 사용자
- 콘텐츠 마케팅 초보자, 상품 제목을 빠르게 만들고 싶은 1인 판매자/글쓰기 초보자
- 1인 사용 (관리자 기능 없음)

---

## 2. 사용자 여정 (전체 흐름)

**주인공:** 지은, 29세, 스마트스토어 판매자. 무릎보호대를 팔고 있는데 클릭율 높은 제목을 빠르게 만들고 싶다.

### Phase 1 — 데이터 수집 (Steps 1–4)

| 단계 | 유형 | 내용 |
|------|------|------|
| Step 1 | 데이터 기입 | 콘텐츠 텍스트 입력 박스에 글 내용 입력 |
| Step 2 | 데이터 기입 | 팔고자 하는 상품명 입력 |
| Step 3 | 안내 문구 + 링크 | "네이버 데이터랩의 쇼핑 인사이트에서 타겟의 성별과 나이대를 직접 검색해주세요. 꼭 마케팅 콘텐츠 아니어도 좋습니다. 콘텐츠 검색자의 특성을 파악해보세요" + 데이터랩 팝업 링크 |
| Step 4 | 데이터 기입 | 타겟 성별, 나이대 입력 |

### Phase 2 — 타겟 분석 보고서 (Steps 5–6)

| 단계 | 유형 | 내용 |
|------|------|------|
| Step 5 | Claude API 호출 #1 | 상품명에 대한 마케팅 타겟 분석 보고서 요약 요청 |
| Step 6 | 데이터 호출 | 보고서 요약 결과를 화면 텍스트박스에 표시 |

### Phase 3 — 제목 생성 (Steps 7–8)

| 단계 | 유형 | 내용 |
|------|------|------|
| Step 7 | Claude API 호출 #2 | Steps 1–6 데이터 전체를 기반으로 제목 생성 요청. 자동 추가 문구 포함 |
| Step 8 | 데이터 호출 | 생성된 제목을 화면 텍스트박스에 표시 |

**Step 7 자동 추가 시스템 프롬프트:**
> "핵심 상품명이 제목 맨 앞쪽에 나올수록 좋다. 제목은 15글자 미만으로 쓰기."
> "타겟의 특성이 일치하는 SNS 유행 콘텐츠의 어조 쓰기"

### Phase 4 — 제목 차별화 (Steps 9–10)

| 단계 | 유형 | 내용 |
|------|------|------|
| Step 9 | Claude API 호출 #3 | 생성된 제목을 네이버 상위노출 콘텐츠와 차별화 요청 |
| Step 10 | 데이터 호출 | 차별화된 제목을 화면에 표시 |

**Step 9 자동 추가 시스템 프롬프트:**
> "위 제목을 차별화해줘. 기준은 네이버 상위노출 페이지에 나오는 콘텐츠 제목과 달라야 돼"

### Phase 5 — 피드백 루프 (Steps 11–15)

| 단계 | 유형 | 내용 |
|------|------|------|
| Step 11 | 안내 문구 + 입력 | "개선하고 싶은 부분을 쓰세요" 텍스트 입력 박스 |
| Step 12 | Claude API 호출 #4 (조건부) | 사용자 개선 피드백 반영하여 제목 재요청 |
| Step 13 | 체크박스 | "더 수정하고 싶으세요?" YES / NO 선택 |
| Step 14 | YES 분기 | 개선 사항 입력 → Claude API 재호출 (Step 11로 루프) |
| Step 15 | NO 분기 | "제목이 완성되었습니다!" 완료 문구 표시 |

## 3. 기능 요구사항

### FR-01 콘텐츠 텍스트 입력 박스 (Step 1)
- 글 내용, 주제, 키워드를 자유롭게 입력하는 멀티라인 텍스트 영역
- 입력 내용은 세션 내 유지

### FR-02 상품명 입력 박스 (Step 2)
- 단일 라인 텍스트 입력 박스
- 이후 모든 Claude API 호출의 핵심 컨텍스트로 사용

### FR-03 네이버 데이터랩 안내 문구 + 링크 (Step 3)
- 고정 안내 문구 표시:
  > "네이버 데이터랩의 쇼핑 인사이트에서 타겟의 성별과 나이대를 직접 검색해주세요. 꼭 마케팅 콘텐츠 아니어도 좋습니다. 콘텐츠 검색자의 특성을 파악해보세요"
- 링크 버튼: `https://datalab.naver.com/shoppingInsight/sCategory.naver` → 새 탭으로 열림

### FR-04 타겟 특성 입력 (Step 4)
- 성별 선택 (남성 / 여성 / 무관) [ASSUMPTION: 드롭다운 또는 라디오 버튼]
- 나이대 입력 (예: 30대, 40–50대 등) [ASSUMPTION: 자유 텍스트 또는 선택지]

### FR-05 Claude API 호출 #1 — 타겟 분석 보고서 (Step 5–6)
- 입력: 상품명 (FR-02)
- 백엔드가 시스템 프롬프트와 결합하여 Claude API 호출
- 출력: 마케팅 타겟 분석 보고서 요약 → 화면 텍스트박스에 표시
- 로딩 중 진행 표시 제공

### FR-06 Claude API 호출 #2 — 제목 생성 (Step 7–8)
- 입력: Steps 1–6 전체 데이터 (콘텐츠, 상품명, 타겟 특성, 보고서 요약)
- 자동 추가 시스템 프롬프트:
  - "핵심 상품명이 제목 맨 앞쪽에 나올수록 좋다. 제목은 15글자 미만으로 쓰기."
  - "타겟의 특성이 일치하는 SNS 유행 콘텐츠의 어조 쓰기"
- 출력: 제목 후보 → 화면 텍스트박스에 표시

### FR-07 Claude API 호출 #3 — 제목 차별화 (Step 9–10)
- 입력: 이전 대화 컨텍스트 유지 (Step 6, 8 결과 포함)
- 자동 추가 시스템 프롬프트:
  - "위 제목을 차별화해줘. 기준은 네이버 상위노출 페이지에 나오는 콘텐츠 제목과 달라야 돼"
- 출력: 차별화된 제목 → 화면에 표시

### FR-08 피드백 입력 + 재생성 루프 (Steps 11–14)
- "개선하고 싶은 부분을 쓰세요" 텍스트 입력 박스 항시 표시
- 사용자 피드백 입력 후 재요청 시 Claude API 추가 호출
- 루프 횟수 제한 없음 [ASSUMPTION: 과도한 호출 방지 위해 추후 제한 고려]

### FR-09 완료 체크박스 + 완료 처리 (Steps 13–15)
- "더 수정하고 싶으세요?" YES / NO 체크박스
- NO 선택 시: "제목이 완성되었습니다!" 완료 문구 표시
- YES 선택 시: 피드백 입력으로 루프 (FR-08)
